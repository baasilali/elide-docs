# Compatibility

Overview of language compatibility and standards supported by Elide.

## JavaScript/TypeScript

### ECMAScript Support

Elide supports ES2024+ with the following features:

- ✅ Classes and modules
- ✅ Async/await
- ✅ Destructuring
- ✅ Template literals
- ✅ Arrow functions
- ✅ Promises
- ✅ Generators
- ✅ Proxies and Reflect
- ✅ BigInt
- ✅ Optional chaining (`?.`)
- ✅ Nullish coalescing (`??`)
- ✅ Top-level await

### TypeScript

- **Version**: 5.0+
- **Features**: All TypeScript language features
- **Compilation**: Native TypeScript support

## Python

### Version Support

- **Python 3.11+**
- **CPython-compatible**

### Standard Library

Most of the Python standard library is supported:

- ✅ `os`, `sys`, `pathlib`
- ✅ `json`, `csv`, `xml`
- ✅ `re`, `datetime`, `time`
- ✅ `collections`, `itertools`
- ✅ `math`, `random`, `statistics`
- ✅ `urllib`, `http.client`
- ⚠️ Limited: `threading`, `multiprocessing`

### Popular Packages

| Package | Status |
|---------|--------|
| NumPy | ✅ Supported |
| Pandas | ✅ Supported |
| Requests | ✅ Supported |
| Flask | ⚠️ Partial |
| Django | ❌ Not yet |

## Known Limitations

### All Languages

- **Native extensions**: Limited support for C/C++ extensions
- **Threading**: Some threading features may have limitations
- **JNI**: Java Native Interface not available in native image

### JavaScript

- **npm packages**: Some packages with native dependencies may not work
- **Specific modules**: `child_process`, `cluster` have limitations

### Python

- **C extensions**: Pure Python preferred
- **Threading**: GIL may affect multi-threaded code
- **pip packages**: Some packages require additional configuration


## Migration Tips

### From Node.js

Most Node.js code works with minimal changes:

```javascript
// Node.js
const fs = require('fs')

// Elide (use ES modules)
import fs from 'fs'
```

### From CPython

Most Python code works as-is:

```python
# Same code works in Elide
import json

data = {'key': 'value'}
print(json.dumps(data))
```
