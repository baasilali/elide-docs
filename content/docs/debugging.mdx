# Debugging

Learn how to debug applications in Elide

This exercise is part of the Polyglot 101 intro guide.

Elide provides multiple ways to debug your code, from simple console logging to full Chrome DevTools integration.

## Before you start

Before running the code below, you'll need to make sure the following is true:

- Elide is installed
- The `elide` binary is executable
- The `elide` binary is on your `PATH`, or otherwise referenced in absolute form

## Console Logging

The simplest way to debug is using console output:

```javascript
// Basic logging
console.log('Debug message:', someValue)
console.error('Error:', error)
console.warn('Warning:', warning)

// Inspect objects
const user = { name: 'Alice', age: 30 }
console.log('User:', user)
```

In Python:

```python
# Basic logging
print('Debug message:', some_value)

# Inspect objects
user = {'name': 'Alice', 'age': 30}
print('User:', user)
```

## Using the Debugger

Elide supports the Chrome DevTools Protocol for interactive debugging.

1. Start your app with the inspector enabled:

```bash
elide run --inspect app.js
```

2. Open Chrome and navigate to `chrome://inspect`

3. Click "Configure" and add `localhost:9229` if not already present

4. Click "inspect" on your Elide process

You can now:
- Set breakpoints
- Step through code
- Inspect variables
- View the call stack

## Debugger Statement

Add breakpoints directly in your code:

```javascript
function calculateTotal(items) {
  debugger  // Execution will pause here
  return items.reduce((sum, item) => sum + item.price, 0)
}
```

The debugger statement only works when running with `--inspect`.

## REPL Debugging

Use the interactive REPL to test code snippets:

```bash
elide repl
elide (js) > const data = [1, 2, 3, 4, 5]
elide (js) > data.map(x => x * 2)
[2, 4, 6, 8, 10]
```

This is useful for:
- Testing functions quickly
- Exploring APIs
- Validating assumptions

## Common Debugging Tasks

### Check variable values

```javascript
const result = someFunction()
console.log('Result:', result)
```

### Trace execution flow

```javascript
function processData(data) {
  console.log('Processing started')
  const cleaned = cleanData(data)
  console.log('Data cleaned:', cleaned)
  const validated = validateData(cleaned)
  console.log('Data validated:', validated)
  return validated
}
```

### Catch errors

```javascript
try {
  const result = riskyOperation()
  console.log('Success:', result)
} catch (error) {
  console.error('Error occurred:', error.message)
  console.error('Stack:', error.stack)
}
```
