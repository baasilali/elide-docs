# Elide Runtime

Use Elide to run JavaScript, Python, Ruby, and more in a single, blazingly fast polyglot runtime.

## What is the Elide Runtime?

The Elide Runtime is a **polyglot execution environment** that allows you to write applications in multiple programming languages that seamlessly interoperate within a single process. Unlike traditional approaches that run separate language runtimes and communicate via serialization, Elide provides a unified runtime powered by GraalVM where languages can directly call each other's functions, share data structures, and work together efficiently.

## Why Elide Excels

### True Polyglot Architecture

- **One Process**: No subprocess orchestration or IPC overhead
- **One Interpreter**: Unified GraalVM Truffle-based execution
- **Zero Serialization**: Share objects and functions directly between languages
- **Seamless Interop**: Import Python from JavaScript, Ruby from Python, etc.

### Performance & Efficiency

- **JIT Compilation**: Fast execution after warmup with GraalVM's optimizing compiler
- **Native Image**: Compile to standalone native binaries with instant startup
- **Low Memory Footprint**: Shared runtime reduces memory overhead
- **World-Class Benchmarks**: Independently benchmarked by TechEmpower

### Modern Standards

- **Web APIs**: Fetch, Crypto, Encoding, URL, and more
- **Node.js Compatibility**: Extensive Node API support
- **WinterTC Conformance**: Aligned with WinterCG standards
- **Built-in Tooling**: Includes oxc, orogene, uv, and Pkl

### Developer Experience

- **Interactive REPL**: Test code instantly in any language
- **Hot Reload**: Watch mode for rapid development
- **Debugging Support**: Step through code across language boundaries
- **Rich Diagnostics**: Clear error messages and stack traces

## CLI Commands

Elide provides a comprehensive command-line interface for running scripts, managing projects, building native binaries, and more.

### Quick Start

The simplest way to run code with Elide:

```shell
$ elide app.js
$ elide script.py
$ elide program.rb
```

Elide automatically detects the language from the file extension (`.js`, `.ts`, `.py`, `.rb`, `.kts`, etc.).

## Architecture

Elide is powered by several cutting-edge technologies:

### Core Technologies

**GraalVM & Truffle**

- <a href="https://graalvm.org" target="_blank" rel="noopener noreferrer" class="inline-flex items-center gap-0.5 text-primary hover:text-primary/80 transition-colors border-b border-primary/40 hover:border-primary/60" style="text-decoration: none;">GraalVM<svg class="h-3 w-3 inline" xmlns="http://www.w3.org/2000/svg" fill="none" viewBox="0 0 24 24" stroke="currentColor" stroke-width="2"><path stroke-linecap="round" stroke-linejoin="round" d="M10 6H6a2 2 0 00-2 2v10a2 2 0 002 2h10a2 2 0 002-2v-4M14 4h6m0 0v6m0-6L10 14" /></svg></a> is Elide's main execution engine and interpreter/compiler
- Leverages <a href="https://www.graalvm.org/latest/graalvm-as-a-platform/language-implementation-framework/" target="_blank" rel="noopener noreferrer" class="inline-flex items-center gap-0.5 text-primary hover:text-primary/80 transition-colors border-b border-primary/40 hover:border-primary/60" style="text-decoration: none;">Truffle<svg class="h-3 w-3 inline" xmlns="http://www.w3.org/2000/svg" fill="none" viewBox="0 0 24 24" stroke="currentColor" stroke-width="2"><path stroke-linecap="round" stroke-linejoin="round" d="M10 6H6a2 2 0 00-2 2v10a2 2 0 002 2h10a2 2 0 002-2v-4M14 4h6m0 0v6m0-6L10 14" /></svg></a> language engines including GraalJS, GraalPython, TruffleRuby, and Sulong

**Kotlin & JVM**

- Most of Elide is written in <a href="https://kotlinlang.org" target="_blank" rel="noopener noreferrer" class="inline-flex items-center gap-0.5 text-primary hover:text-primary/80 transition-colors border-b border-primary/40 hover:border-primary/60" style="text-decoration: none;">Kotlin<svg class="h-3 w-3 inline" xmlns="http://www.w3.org/2000/svg" fill="none" viewBox="0 0 24 24" stroke="currentColor" stroke-width="2"><path stroke-linecap="round" stroke-linejoin="round" d="M10 6H6a2 2 0 00-2 2v10a2 2 0 002 2h10a2 2 0 002-2v-4M14 4h6m0 0v6m0-6L10 14" /></svg></a> and built against JVM 23
- Null safety and memory safety by default

**Rust**

- Native portions written in <a href="https://rustlang.org" target="_blank" rel="noopener noreferrer" class="inline-flex items-center gap-0.5 text-primary hover:text-primary/80 transition-colors border-b border-primary/40 hover:border-primary/60" style="text-decoration: none;">Rust<svg class="h-3 w-3 inline" xmlns="http://www.w3.org/2000/svg" fill="none" viewBox="0 0 24 24" stroke="currentColor" stroke-width="2"><path stroke-linecap="round" stroke-linejoin="round" d="M10 6H6a2 2 0 00-2 2v10a2 2 0 002 2h10a2 2 0 002-2v-4M14 4h6m0 0v6m0-6L10 14" /></svg></a> for maximum safety outside the JVM

**Networking**

- Powered by <a href="https://netty.io" target="_blank" rel="noopener noreferrer" class="inline-flex items-center gap-0.5 text-primary hover:text-primary/80 transition-colors border-b border-primary/40 hover:border-primary/60" style="text-decoration: none;">Netty<svg class="h-3 w-3 inline" xmlns="http://www.w3.org/2000/svg" fill="none" viewBox="0 0 24 24" stroke="currentColor" stroke-width="2"><path stroke-linecap="round" stroke-linejoin="round" d="M10 6H6a2 2 0 00-2 2v10a2 2 0 002 2h10a2 2 0 002-2v-4M14 4h6m0 0v6m0-6L10 14" /></svg></a> for world-class performance and durability
