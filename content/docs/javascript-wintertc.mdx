# WinterTC

<a href="https://wintertc.org/" style="color: rgb(168, 85, 247); text-decoration: underline; font-weight: 500;">WinterTC</a> (AKA ECMA International Technical Committee 55) is an open collaborative group committed to standardizing web-facing JavaScript runtimes; "web-interoperable" describes a package of new standards and definitions:

- **<a href="https://min-common-api.proposal.wintertc.org/" style="color: rgb(168, 85, 247); text-decoration: underline; font-weight: 500;">Minimum Common API</a>**: Defines a standard set of "minimum common APIs" which all web-interoperable runtimes should support.

- **<a href="https://sockets-api.proposal.wintertc.org/" style="color: rgb(168, 85, 247); text-decoration: underline; font-weight: 500;">Sockets API</a>**: Defines APIs for communicating over custom TCP sockets from non-browser JavaScript applications.

- **<a href="https://github.com/wintercg/proposal-cli-api" style="color: rgb(168, 85, 247); text-decoration: underline; font-weight: 500;">CLI API</a>**: Defines standards for accessing system environment, invocation arguments, and other CLI-related APIs.

## Elide + WinterTC

Elide aims to conform to most WITC standards:
