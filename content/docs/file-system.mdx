# File System

Filesystem access with Elide

This exercise is part of the Polyglot 101 intro guide.

Coming soon.

<Alert>
  <strong>Note:</strong> Like <a href="/docs/env-variables.html">env variables</a>, Elide <strong>isolates filesystem I/O by default</strong>; unless you grant permission, code running under Elide (in any language) cannot see your disk.
</Alert>

## Before you start

Before running the code below, you'll need to make sure the following is true:

- Elide is installed
- The `elide` binary is executable
- The `elide` binary is on your `PATH`, or otherwise referenced in absolute form

## Reading files from disk

We're going to try out Elide's isolation with regard to **disk I/O**

1. Start an interactive session. Elide starts you with JavaScript. Let's also create a file which we will read:

```bash
echo "testing" > hello.txt
elide repl
elide (js) >
```

Use `Ctrl+C` twice or `Ctrl+D` to exit the interactive session.

2. Now, let's import the [`node:fs`](https://nodejs.org/api/fs.html) module:

```bash
elide (js) > const fs = require('node:fs');
```

Elide aims for full Node API support, but we aren't there yet. See the Node API reference.

3. Let's read the file we just wrote:

```bash
elide (js) > fs.readFileSync("hello.txt", {encoding: 'utf-8'})
testing
```

Neat! We can read the file.
