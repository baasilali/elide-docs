# Welcome to Elide

Elide is a high-performance polyglot runtime built on GraalVM that lets you run JavaScript, TypeScript, Python, and JVM languages in a single application with native-image compilation for instant startup times and minimal memory footprint.

## What is Elide?

Elide combines the best of multiple worlds:

- **Polyglot Runtime**: Run multiple languages in a single application with seamless interoperability
- **Native Compilation**: Compile to native binaries for instant startup (< 10ms) and low memory usage
- **Built-in HTTP Server**: High-performance server powered by Netty (800K+ req/s on Linux)
- **Modern APIs**: Clean, standards-based APIs similar to Deno and Bun
- **GraalVM Foundation**: Built on battle-tested GraalVM technology for optimal performance

## Quick Example

Here's a simple HTTP server in Elide:

```javascript
// server.js
const app = Elide.http

app.router.handle("GET", "/", (request, response) => {
  response.send(200, "Hello, Elide!")
})

app.router.handle("GET", "/api/data", (request, response) => {
  response.json({
    message: "Fast polyglot runtime",
    timestamp: Date.now()
  })
})
```

Run it:

```bash
elide run server.js
```

Compile to native binary:

```bash
elide build --native
```

This creates a standalone binary with no runtime dependencies!

## Why Choose Elide?

### For Web Applications
- Ultra-fast server-side rendering (SSR)
- Minimal resource usage for cost savings
- Single binary deployment
- Built-in HTTP/2 and HTTP/3 support

### For Microservices
- Instant startup for serverless/FaaS
- High throughput HTTP server
- Native image packaging
- Cross-language service composition

### For CLI Tools
- Fast startup makes CLI tools snappy
- No runtime installation required
- Single binary distribution
- Cross-platform support

### For Serverless Functions
- Sub-10ms cold starts eliminate warmup
- Minimal memory reduces costs
- Native binaries work everywhere
- No container overhead

## Elide as a Framework

Elide isn't just a runtime—it's also a modern framework with modular components:

| Module | Description | Platforms |
|--------|-------------|-----------|
| `core` | Essential utilities and APIs | JVM, JavaScript, Native, WASM |
| `base` | Extended functionality | JVM, JavaScript, Native |
| `server` | HTTP server and routing | JVM, Native Image |
| `ssr` | Server-side rendering | JVM, JavaScript |
| `graalvm` | GraalVM integration | JVM, Native Image |

Choose only the modules you need—everything works together seamlessly.

## Language Support

### JavaScript/TypeScript
- **Full ES2024+ support** with modern features
- **V8-compatible** APIs for familiarity
- **TypeScript** with type checking and compilation
- **Top-level await** and modern module system

### Python
- **Native Python execution** via GraalVM
- **NumPy, Pandas** and scientific computing libraries
- **Seamless JS-Python interop** - call Python from JavaScript
- **Full CPython compatibility**

### JVM Languages
- **Java, Kotlin, Scala** first-class support
- **Full Java ecosystem** - use any JVM library
- **Interop with guest languages**
- **Native image compilation**

## Project Philosophy

Elide is designed around three core principles:

1. **Performance First**: Native compilation, advanced JIT, zero-copy operations
2. **Developer Experience**: Clean APIs, excellent tooling, great documentation
3. **Flexibility**: Use the best language for each task, compose seamlessly

## Use Cases

Elide excels at:

- **High-Performance APIs**: Low latency, high throughput web services
- **Serverless Functions**: Instant cold starts, minimal memory
- **Full-Stack Applications**: SSR, API, database—all in one runtime
- **CLI Tools**: Fast, distributable command-line applications
- **Microservices**: Efficient resource usage for dense deployments
- **Edge Computing**: Small footprint for edge/CDN deployment

## What's Included

Out of the box, Elide provides:

- **HTTP Server**: High-performance Netty-based server
- **Module System**: ES modules, CommonJS, native imports
- **File System**: Standards-based file operations
- **Crypto**: Built-in cryptographic primitives
- **Fetch API**: Standard HTTP client
- **Console & Logging**: Structured logging support
- **Environment**: Access to env vars and system info
